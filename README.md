# m-wap-pages

## 项目简介

m-wap-pages 是一个用于归档和发布多个独立 HTML 页面的仓库。  
该仓库不针对任何特定网站或域名，仅作为静态页面内容的集合与展示用途。  
适合存放移动端适配的页面、轻量级工具页面、演示页面等。

## 目录说明

```
m-wap-pages/
├── pages/          # 存放各个独立 HTML 页面
├── assets/         # 共享资源文件（CSS、JS、图片等）
├── index.html      # 页面索引或入口
├── README.md       # 本文件
└── ...
```

- `pages/` 目录下每个子文件夹或文件代表一个独立页面。
- `assets/` 用于存放多个页面共用的静态资源，避免重复。

## 页面归档说明

本仓库中的页面为独立归档，彼此之间无功能依赖。  
每个页面均可通过直接访问 HTML 文件路径查看效果。  
页面内容可能包含演示数据、示例样式、静态交互等，不涉及真实业务数据。

归档的页面类型包括但不限于：
- 移动端适配的静态页面
- 轻量级功能演示页
- 临时页面存档
- 其他独立 HTML 页面

## 维护说明

- 添加新页面：在 `pages/` 下创建新目录，放入 `index.html` 及相关资源。
- 更新页面：直接修改对应的 HTML 或资源文件。
- 删除页面：移除对应目录及引用。
- 提交前建议检查页面完整性及资源路径正确性。

欢迎维护者根据实际需要调整目录结构或命名规范。

## 说明

- 本仓库不收集用户数据，不包含任何跟踪代码。
- 页面内容仅供展示与归档，不构成任何形式的服务承诺。
- 所有页面均以静态形式存在，无需后端支持。
